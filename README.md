# koyomi-score

`koyomi-score` は、日付と用途を選ぶと、その日の暦注をもとにした独自スコアと過ごし方のメモを表示する暦アプリです。

六曜、十二直、日干支、九星、選日などを「今日どう動くか」の参考にしやすい形へ整理します。

## 現在の構成

- `index.html`: 画面、スタイル、暦計算、スコア計算、デバッグ出力を含む単体ファイル
- `tests/koyomi-verified.test.js`: 外部確認済み日付の簡易検算テスト
- `README.md`: プロジェクト概要と仕様メモ
- `CHANGELOG.md`: 変更履歴
- `.nojekyll`: GitHub PagesでJekyll処理を使わず静的ファイルとして配信するための空ファイル

ビルド環境や外部ライブラリは使っていません。ブラウザで `index.html` を直接開いて動作します。

## デザイン方針

毎日安心して見られる暦アプリとして、白から生成り系の背景、淡いクリーム系のカード、山吹系のメインカラーを基調にしています。

CSSの `:root` に、配色、余白、角丸のデザイントークンを定義しています。

- `--color-bg`
- `--color-surface`
- `--color-primary`
- `--color-primary-soft`
- `--color-good`
- `--color-caution`
- `--color-text`
- `--color-muted`
- `--radius-card`
- `--radius-pill`
- `--space-xs`
- `--space-sm`
- `--space-md`
- `--space-lg`

良い日、注意日、休息向きなどの色は、強く出すぎない淡い色に調整しています。注意系も赤を強くせず、薄いオレンジやくすみ朱色系で表示します。

## 対応範囲

- 対応日付: `1940-01-01` から `2099-12-31`
- 既定の日付: 今日の日付。範囲外の場合は `2026-05-16`
- 表示言語: 日本語
- 旧暦取得: `Intl.DateTimeFormat("ja-JP-u-ca-chinese")` に依存

旧暦を取得できないブラウザでは、暦データなしのメッセージを表示します。

## 主要機能

- 日付を指定して暦情報を表示
- 「今日やりたいこと」を選び、用途別の暦スコアを表示
- 画面上部の「今日のテーマ」で、その日の過ごし方の方向性を表示
- カテゴリスコア上位3件をおすすめカテゴリとして表示
- テーマの直下に今日の暦を表示し、判断根拠を確認しやすくする
- 主要カテゴリのスコアを常時表示し、クリックで用途を切り替え
- 月間カレンダーで、月全体の総合スコア分布を表示
- 現在の用途について、未来365日以内の次のおすすめ日を表示
- 未来ランキングで、指定期間内のおすすめ日・避けたい日TOP10を表示
- 今日のスコア、ランキング、候補日に主な加点要素と注意要素を同じ表示ルールで表示
- スコア理由は上位3件まで表示し、残りは折りたたみで表示
- よく使う用途と、詳細用途の2段階選択
- 総合スコア、テーマ、日ごとのコメントを表示
- おすすめ行動は3件まで表示し、残りは折りたたみで表示
- 注意点を表示
- 詳細表示で暦注ごとの判定根拠を表示
- 検算用デバッグ表とCSVを表示
- 医療系用途では注意メッセージを表示

## スコア計算の概要

スコアは100点満点の独自指標です。

1. 基礎点を `60` とする
2. 暦注ごとの重みを用途カテゴリに応じて加算・減算する
3. 用途の方向性に合う暦注を加算・減算する
4. 内部用の `rawScore` を保持する
5. 表示用スコアは `0` から `100` の範囲に丸める
6. 特定条件では上限をかける

未来ランキングの並び替えでは、表示用の100点丸めスコアではなく、内部の `rawScore` に上限制御を反映した値を使います。これにより、表示上は同じ100点の日でも順位差が出ます。

ランキングの納得感を調整するため、暫定的に以下のバランスで重みを置いています。

- 大安: 日常的な吉日として控えめに評価
- 友引: 対人関係に強い特化型の吉日として評価
- 仏滅: 赤口より重く、受死日・十死日より軽い一般的な凶日として評価
- 赤口: 火や刃物などの注意日だが、総合では仏滅より軽く評価
- 一粒万倍日: 目的次第で強い吉日として評価
- 天赦日: 特別な大吉日だが、単独で無条件に上位固定になりすぎないよう調整
- 神吉日: 出現頻度を考慮して補助的な吉日として評価
- 月徳日、甲子の日: レア度と暦上の意味を考慮して少し強めに評価
- 凶会日: 強い凶日群より軽めの注意日として評価

現時点では、以下の上限制御を適用しています。

- 受死日: 用途別上限を適用。総合は `65`、開始・契約・大きい買い物系は `58`、医療・身体寄りは `50`、休息・整理系は `78`
- 十死日: 上限 `52`
- 不成就日: 契約、開始、大きい買い物系では上限 `62`
- 三隣亡: 引越し用途では上限 `45`

## 月間カレンダー

月単位で総合スコアを俯瞰するカレンダーを表示します。

- 表示対象: 総合スコア
- 操作: 前月 / 次月
- 日付クリック: 選択した日付を現在の結論カードへ反映
- 表示形式: 日曜始まり
- 今日の日付: 枠線で強調
- 選択中の日付: 濃い枠線で強調
- 曜日ヘッダー: 日曜を赤系、土曜を青系で表示

スコア帯ごとに以下の色で表示します。

- `90` から `100`: 緑
- `75` から `89`: 水色
- `60` から `74`: 白
- `40` から `59`: 薄オレンジ
- `0` から `39`: 赤系

カテゴリ別カレンダー表示は将来対応予定です。

## 未来ランキング

未来の日付を目的別に並べるランキングを表示します。

### 次のおすすめ日カード

現在選択している用途について、未来365日以内で最初に出現する90点以上の日を表示します。表示項目は日付、スコア、あと何日、主な加点要素2件です。

90点以上の日が見つからない場合は、未来365日以内で最もスコアが高い日を参考候補として表示します。

- 目的: 現在選択している「今日やりたいこと」を使用
- 期間: 30日 / 90日 / 180日 / 365日
- 種別: おすすめ日 / 避けたい日
- 対象: 選択中の日付の1日後から指定期間内
- 件数: TOP10
- 並び順: 内部の順位用スコアを使用

各行には、順位、日付、スコア、主な加点要素、主な減点要素、一言コメントを表示します。

## 判定ラベル

スコアに応じて以下のラベルを表示します。

- `85` 以上: 追い風の日
- `70` 以上: 整えながら進む日
- `55` 以上: 無理なく進める日
- `40` 以上: 守りを意識する日
- `40` 未満: 静かに整える日

医療系用途では、スコアにかかわらず「暦の参考メモ」として表示します。

## 医療系用途の扱い

以下の用途では、暦を医療判断として扱わない注意文を表示します。

- 注射・採血
- 病院・検査
- 鍼灸・整体

医療の日程や治療内容は、必ず医師・医療機関の判断を最優先します。

## 検算状況

`index.html` 内に `debugDates2026` と `verificationResults` があり、2026年の一部日付について外部サイトとの照合メモを残しています。

神吉日は干支番号ではなく干支名ベースの一覧で管理しています。既知の差分があった `2026-01-01`、`2026-03-05`、`2026-06-29` と、神吉日ありの `2026-12-22` は `tests/koyomi-verified.test.js` で検算しています。

Node.jsが使える環境では、以下で簡易検算できます。

```sh
node tests/koyomi-verified.test.js
```

現時点で主要項目はおおむね照合済みですが、選日一覧には差分が残っています。特に以下は今後の確認対象です。

- 神吉日の検算対象日をさらに増やす
- 凶会日・月破が外部確認結果より多く出る日がある
- 天火日は未収録
- 大土・小土などの名称差分がある
